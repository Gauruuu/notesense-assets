1.1 Introduction to DBMS

A Database Management System (DBMS) is a software that helps user to manage, store and retrieves data efficiently in a structural format.

  • It allows user to create, update and query databases efficiently.
  • It ensures data integrity, consistency and security across multiple users and applications.
  • It reduces data redundancy and inconsistency through centralized control.
  • It supports concurrent data access, transaction management & automatic backup.

1.2 File System Vs. DBMS

DBMS eliminates drawback data redundancy, inconsistency, difficult access, poor security, single user access, no backup or recovery.

1.3 Data Models – Relational, Hierarchical, Network

Data Models are conceptual tools or blueprints that defines the logical structure of a database. It specifies how the data is connected, stored, accessed and constrained within the system.

RDBMS

  • Best For: Applications requiring high data integrity and complex querying across multiple tables.
  • Mechanism: Uses Primary and Foreign Keys to link tables.
  • Key Advantage: Offers flexibility without rewriting application code.
  • 2026 Examples: MySQL, PostgreSQL, Oracle Database, Microsoft SQL Server.

HDBMS

  • Best For: Simple fixed one-to-many data.
  • Mechanism: Inverted tree structure.
  • Key Advantage: Extremely fast retrieval.
  • 2026 Examples: IBM IMS, Windows Registry.

NDBMS

  • Best For: Complex many-to-many data.
  • Mechanism: Pointer-based graph.
  • Key Advantage: High performance.
  • 2026 Examples: RaimaDB, IDMS.

1.4 Levels of Abstraction

  • Physical Level: Physical storage details.
  • Logical Level: Database structure.
  • View Level: User-specific views.

1.5 Data Independence

Data Independence is the ability to change schema at one level without affecting higher levels.

  • Physical Data Independence
  • Logical Data Independence

Database Languages

DML — SELECT, INSERT, UPDATE, DELETE
DCL — GRANT, REVOKE
TCL — COMMIT, ROLLBACK, SAVEPOINT
